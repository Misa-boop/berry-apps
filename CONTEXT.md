# 校园生活服务智能体 - 领域词汇表

## 核心概念

### 自建业务系统
独立开发的前后端系统，基于 RuoYi（若依）框架二次开发。
- **前端（管理员）**：基于 ruoyi-ui（Vue 3 + TypeScript + Element Plus + Vite），PC端管理后台
- **前端（学生/老师）**：Vue 3 + TypeScript + Element Plus + Vite，响应式布局适配移动端
- **后端**：基于 RuoYi-Vue（SpringBoot 3 + MyBatis Plus + JDK 17），多模块架构
- **AI引擎**：Python FastAPI + LangChain + LangGraph，自建智能体
- **数据库**：MySQL + Redis + FAISS

### 角色
系统中定义的三种用户身份，决定可执行的操作范围：
- **学生**：查询课表、签到、预约图书馆座位、查看校园信息、食堂推荐、信誉分兑换
- **老师**：查询课表、发起签到、预约图书馆座位（拥有信誉分）、查看校园信息
- **管理员**：管理信誉分、管理用户、管理校园数据、维护课表。管理员内部通过权限细分：
  - **系统管理员**：全模块权限
  - **食堂管理员**：仅限食堂模块菜单数据维护权限

### 校园生活智能体
基于 FastAPI + LangChain 自建的AI智能体，是整个系统的"AI大脑"，负责理解用户自然语言意图并调度相应工具/工作流完成服务。单智能体 + 多工具架构。

### 课表
描述课程安排的数据实体。数据来源为管理员手动录入或Excel批量导入。课表模型特征：
- 一门课可有多个上课时间段（如周一1-2节 + 周三3-4节）
- 支持单周/双周课表（odd/even week pattern）
- 每个学生拥有独立课表（非班级共享）

### 签到
课堂出勤确认行为。由老师主动发起，学生在限定时间内完成确认。与课表和课程关联。详细规则：
- **发起方式**：系统根据课表自动推荐当前课程，老师也可手动选择（调课/补课场景）
- **时间窗口**：默认5分钟，老师可修改；老师可手动提前结束签到
- **签到状态**：已签到 → 迟到 → 缺勤（三态流转）。超时未签到自动标记缺勤，老师可手动修改状态（如请假）
- **验证方式**：老师发起时选择验证模式：①简单点击签到 ②位置验证 ③验证码
- **信誉分影响**：签到记录影响信誉分（缺勤扣分）

### 签到状态
签到结果的三种状态：
- **已签到**：在时间窗口内完成签到
- **迟到**：超过时间窗口但在下课之前签到，若下课前老师未手动更改则自动变为缺勤
- **缺勤**：超时未签到，自动标记

### 签到验证模式
老师发起签到时选择的防代签验证方式：
- **简单模式**：学生点击"签到"按钮即可
- **位置验证**：验证学生是否在教室附近
- **验证码模式**：老师展示验证码，学生输入正确验证码完成签到

### 信誉分
贯穿系统的用户信用评分，影响图书馆座位预约权限，也可用于兑换奖励。详细规则：
- **初始值**：10分
- **冻结机制**：预约时冻结1分，正常完成后解冻归还
- **惩罚扣分**：预约未到扣5分，超时占用扣3分
- **奖励加分**：一天内无违规行为奖励1分
- **清零后果**：信誉分为0时无法预约图书馆座位，只能由管理员手动重置

### 图书馆座位预约
图书馆及自习空间的座位预约系统。核心规则：
- **时间段**：上午8-12点、下午14-18点、晚上18-22点
- **预约限制**：同一时间段只能预约一个座位，可连续预约多个时间段；只能提前一天预约
- **座位签到**：预约后到达图书馆需签到确认
- **违规释放**：预约后30分钟未签到则违规自动释放座位
- **临时离开**：更改座位状态（吃饭最长70分钟，上厕所最长20分钟），超时违规自动释放

### 图书馆分区
图书馆及自习空间的物理分区，如图书馆一楼、图书馆二楼、一站式服务社区（书吧）

### 座位类型
座位的设施属性区分：普通座、带电源座、靠窗座

### 座位状态
座位在预约周期内的实时状态：空闲、已预约、使用中、临时离开（吃饭/上厕所）、违规释放中

### 信誉分兑换商城
信誉分的正向激励闭环。用户可用积攒的信誉分兑换小礼物、学校周边、学习资料、书籍等物品

### 食堂餐口
独立于地图的食堂窗口信息系统。收集每个食堂的窗口信息（今日菜单、价格、营业时间），为学生提供个性化推荐。
- **组织层级**：校园 → 食堂 → 楼层 → 窗口 → 菜品
- **菜单维护**：由食堂管理员每日录入，日更模式
- **推荐方式**：简单方案（用户偏好标签规则匹配）+ 智能体方案（自然语言语义匹配）

### 菜品
食堂窗口售卖的食物项。属性：id、名称、价格、分类（荤/素/汤）、口味标签（辣/清淡等）、过敏原标识、上架时间。支持学生评价/评分。

### 食堂管理员
管理员角色下的权限细分，仅负责每日录入和维护所属食堂窗口菜单数据。非独立角色。

### 统一入口
Nginx做统一入口，前端只需对接一个域名。路径规则：`/api/**` → SpringBoot，`/ai/**` → FastAPI。

### SpringBoot（业务主后端，基于若依框架）
基于 RuoYi-Vue 多模块架构二次开发。负责认证 + 全部CRUD + 定时任务 + 业务规则校验：
- **复用若依已有模块**：ruoyi-system（用户认证/JWT/RBAC）、ruoyi-common（通用工具/常量）、ruoyi-framework（框架核心配置）
- **新建 ruoyi-campus 模块**：课表CRUD、签到业务逻辑、图书馆座位预约CRUD、信誉分计算、食堂菜单CRUD、校园地图标注点CRUD、上课提醒（定时任务）、违规自动释放座位（定时任务）

### FastAPI + LangChain（AI智能体引擎）
自建的AI智能体引擎。核心能力：
- **Agent框架**：LangGraph Stateful Agent，支持状态图编排、条件分支、循环
- **底层LLM**：DeepSeek，Function Calling支持好，国内可直接访问
- **向量数据库**：FAISS（开发阶段），后期可迁移到 Milvus
- **工具调用**：LangChain Tool/Function Calling，调用 SpringBoot 业务API
- **SSE流式输出**：FastAPI SSE 推送对话响应
- **RAG知识检索**：LangChain RAG 链路 + FAISS 向量存储
- **对话记忆**：LangGraph 状态管理 + 对话历史持久化

### 智能体工具清单（LangChain Tools）
智能体通过 LangChain Tool 调用 SpringBoot 业务API获取数据：

| 工具名 | 用途 | 对应API |
|--------|------|---------|
| query_schedule | 查询课表 | GET /api/schedule |
| create_checkin | 老师发起签到 | POST /api/checkin |
| do_checkin | 学生签到 | POST /api/checkin/{id}/attend |
| query_seat | 查询座位状态 | GET /api/library/seats |
| reserve_seat | 预约座位 | POST /api/library/reservation |
| seat_checkin | 座位签到 | POST /api/library/reservation/{id}/checkin |
| query_menu | 查询食堂菜单 | GET /api/canteen/menu |
| query_credit | 查询信誉分 | GET /api/credit |
| query_weather | 查询天气 | 第三方天气API |

### 知识库（LangChain RAG）
基于 LangChain RAG 的三个知识库：
- **校园规章制度知识库**：学生手册内容
- **校园新闻/公告知识库**：校园动态信息
- **常见问题FAQ知识库**：高频问答

### 通知提醒
系统主动向用户推送的消息，初期通过站内消息（前端通知栏）实现。触发方式有两种：
- **定时任务提醒**：SpringBoot定时任务，在上课前N分钟（如15分钟）自动推送
- **智能体提醒**：用户对智能体说"提醒我下午3点上课"，智能体设置提醒

### 提醒场景
- **课表提醒**："你下一节课是《高等数学》，地点：教学楼A301，还有15分钟开始"
- **签到提醒**："《高等数学》课程已开始签到，请在5分钟内完成签到"
- **座位预约提醒**："你的图书馆座位预约将在30分钟后开始，请准时签到"

### 天气查询
双通道天气服务：
- **智能体工具**：LangChain Tool 调用第三方天气API，用户问"明天天气怎么样"时智能体调用返回结果
- **前端组件**：首页天气卡片，直接调用第三方天气API展示，不经过智能体
