# Architecture Decision Records (ADR)

This directory contains Architecture Decision Records for the 校园生活服务智能体 project.

## What is an ADR?

An ADR captures an important architectural decision made along with its context and consequences. We create ADRs only when:

1. **Hard to reverse**: The decision is costly to undo once implemented
2. **Surprising without context**: The choice would seem unusual to someone unfamiliar with the project history
3. **Result of a real trade-off**: Multiple valid options existed, and we had to choose between them

## Format

Each ADR follows this structure:

```markdown
# ADR-XXX: Title

## Status
- Proposed / Accepted / Deprecated / Superseded by ADR-YYY

## Context
What is the issue that we're seeing that is motivating this decision or change?

## Decision
What is the change that we're proposing or have agreed to implement?

## Consequences
What becomes easier or more difficult to do because of this change?
```
